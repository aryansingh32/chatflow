## 2026-05-14 - Prevent O(n) re-renders in Message List\n**Learning:** Chat components in React require manual memoization for the individual message items, otherwise all previous messages re-render when a new message is pushed or a typing indicator changes. This causes an O(n) rendering penalty on each keystroke or new chat message.\n**Action:** Use `React.memo` on all map-rendered list items in chat/feed views, specifically prioritizing ones like `MessageItem` that rarely change their internal props.
